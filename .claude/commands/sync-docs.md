현재 디렉토리와 하위 폴더에 있는 소스 코드, 최신 Git 히스토리, 그리고 docs/ 하위의 마크다운 파일들을 꼼꼼하게 스캔하고 분석해 줘. 다음 단계를 순서대로 수행해:

1. CLAUDE.md 파일은 세션 간 컨텍스트를 유지하기 위한 Source of Truth야. 모든 규칙은 CLAUDE.md 파일을 기반으로 프로젝트 계획 및 코드 수정 및 파일 생성, 그리고 평가까지 진행해.
2. Git 및 코드 분석: `git status`, `git log`, `git diff` 등을 활용해 최근 커밋 및 푸쉬 내역, 그리고 아직 커밋되지 않은 작업 내역을 확인해. 이를 바탕으로 최근 변경된 로직과 그 의도, 현재 구현된 핵심 기능 및 아키텍처를 파악해. (node_modules, venv 등 패키지 폴더는 제외)
3. 문서 검토: CLAUDE.md와 아래 docs/ 경로의 모든 마크다운 파일을 읽고, 현재 코드베이스와 최신 Git 변경 사항을 문서가 제대로 반영하고 있는지 확인해.
   - docs/plan/*.md       — 전체 로드맵 및 Phase별 상세 스펙
   - docs/in-progress/*.md — 현재 진행 중인 Step 스펙
   - docs/completed/*.md  — 완료된 Step 기록
   - docs/reference/*.md  — 외부 API·기술 스펙 레퍼런스
4. 교차 검증 (Gap Analysis): 
   - 코드나 Git 내역에는 반영되었으나 문서에는 누락된 부분
   - 문서에는 존재하나 실제 코드에서는 삭제되거나 구조가 변경된 부분
   - 최근 커밋 메시지의 핵심 의도 중 문서화가 필요한 부분
   - docs/in-progress/ 파일 중 이미 완료된 것은 docs/completed/로 이동 필요한지 확인
5. 해결 및 실행: 
   - 발견된 불일치 사항들을 간략히 리스트업 해줘.
   - 마크다운 파일을 최신 상태로 동기화하기 위해 파일을 직접 수정하거나, 필요한 쉘(shell) 명령어를 생성해 줘. 내가 바로 승인(Accept)하고 적용할 수 있도록 만들어 줘.
